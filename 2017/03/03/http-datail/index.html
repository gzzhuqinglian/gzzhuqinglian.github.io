文章目錄

http协议是应用层协议，在这个协议中，需要我们掌握，请求request的相关知识。

作为前端开发，对于http协议的掌握尤其重要。很多人都忽悠这一块，包括我。面试的时候问到这一块的时候，我感觉好生疏。
http的请求头部包含那些东西呢？哪些需要我们去重点掌握

Content-Type 设置请求体的MIME类型（适用POST和PUT请求）,具体的形式是这样的，Content-Type: [type]/[subtype]; parameter。
type的种类如下：
Application：用于传输应用程序数据或者二进制数据；
Message：用于包装一个E-mail消息；
Image：用于传输静态图片数据；
Audio：用于传输音频或者音声数据；
Video：用于传输动态影像数据，可以是与音频编辑在一起的视频数据格式。

subtype用于指定type的详细形式。content-type/subtype配对的集合和与此相关的参数，将随着时间而增长。为了确保这些值在一个有序而且公开的状态下开发，MIME使用Internet Assigned Numbers Authority (IANA)作为中心的注册机制来管理这些值。
常用的组合如下：
multipart/form-data 包含文件等信息。
applicaiton/json json格式，等等。
