前言

任何代码，我们都想尽可能的复用，这里就谈到了模块化的概念，在js的模块定义中，又3个东西需要我们掌握。

1.commonjs 为了解决代码复用的问题，主要用于后端，提出了模块化的概念。
2.amd(异步加载规范)，为了解决浏览器的模块复用的问题，代表requirejs
3.cmd 为了解决浏览器缓慢执行的问题，代表seajs

一:commonjs模块的定义分为：{模块引用(require)，模块定义(exports)，模块标识(module)}
demo如下：
//sum.js

exports.add=function(a,b){
return a+b;
}
//calculate.js

var sum=require(sum);
exports.add=function(a,b){
return sum.add(a,b);
}
上面的代码中，用到了require，和exports。

二.amd主要用于前端代码的复用。
amd异步模块定义。amd只有一个接口，define(id,dependencies，factory).
象这样 define([‘dep1’,’dep2’],function(dep1,dep2){…});

这样依赖前置，有一个问题，当前模块的执行，要等到依赖模块的加载，并且执行完成，这个有效率上的问题。

三.cmd 主要用于前端代码的缺陷。

define(function(require,exports,module){…});

所有的模块的依赖等到用的时候再去用。

具体的可以参考如下的链接：http://www.2cto.com/kf/201411/348276.html
